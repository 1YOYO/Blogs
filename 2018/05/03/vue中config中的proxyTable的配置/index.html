proxyTable?

在开发的时候，接口联调一般都是同域名下，且不存在跨域的情况下进行接口，但是当我们现在使用脚手架的时候，需要本地启动服务后，比如本地开发服务下是 http://localhost:8080 这样的访问页面，但是我们的接口地址是 http://xxxx.com/ 这样的，会存在跨域的请求，因此我们需要在打包的时候配置一下，就用到了 config/index.js 代码下的proxyTable配置：

接口地址原本是 /list，但是为了匹配代理地址，在前面加一个 /rest, 因此接口地址需要写改成这样 /rest/list。

注意： ‘/rest’ 为匹配项，target 为被请求的地址，因为在 this.$http.get() 的 url 中加了前缀 ‘/rest’，而原本的接口是没有这个前缀的，所以需要通过 pathRewrite 来重写地址，将前缀 ‘/rest’ 转为 ‘/‘。如果本身的接口地址就有 ‘/rest’ 这种通用前缀，就可以把 pathRewrite 改成’^/rest’: ‘/rest’或者删掉pathRewrite。
